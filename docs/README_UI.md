# UI 布局建议（面向可交互调参、验证与对比实验）

下面给出一套面向数据科学家与竞赛选手的桌面级 UI 布局方案，目标是：**清晰、可复现、快速试验、避免数据泄露**。布局按视觉层次和使用频率组织——常改的参数放在显眼位置，耗时或危险操作放在高级面板并要求确认。每个区域给出推荐控件类型、默认值建议与交互行为。

---

## 总体结构（窗口级）

- 布局建议：**左侧侧边栏（配置）｜中间主区（运行与可视化）｜右侧侧栏（诊断与对比）**
- 底部：**运行控制栏 + 实验日志 / 运行历史**
- 顶部工具栏（全局）：项目名、上传数据、保存/加载配置、预设下拉、帮助/验证按钮

视觉比例参考：

- 左侧栏：约 20% 宽度（参数树）
- 中间主区：约 55% 宽度（主要交互、图表、训练输出）
- 右侧栏：约 25% 宽度（KPI、诊断、对比）
- 底部：固定高度约 120–200 px（控制栏、进度、运行列表）

设计理由：常见工作流程是在调整参数、运行训练与观察评估结果之间循环，中央用于展示结果与快速看板；右侧用于记录历史与长期对比。

---

## 左侧侧边栏：参数面板（分组、可折叠）

建议将对外暴露的参数按「全局 / 数据」「特征工程」「验证」「模型相关（LightGBM / NN / PPO）」与「后处理」分组，每组作为可折叠面板（类似 accordion）。常用项默认展开，高级项默认折叠。

1. 数据（默认展开，第一位）

   - 文件上传：`train.csv` / `test.csv`（文件选择控件），配合“预览数据”按钮。
   - 缺失值处理下拉：前向填充 / 中位数 / 填 0 / 插值 等
   - 开关：是否添加 `_is_missing` 指示变量

2. 验证（Validation，必看）

   - 验证方法选择：Purged K-Fold（去泄露分折）或 Walk-Forward（滚动前向）
   - 若选择 Purged K-Fold：设置 `n_splits`（折数）、`embargo_days`（禁封天数）、`label_horizon`（标签时长）
   - 若选择 Walk-Forward：设置 `train_window`、`val_window`、`step`
   - 按钮：验证 CV / 分数一致性（运行小样本检验并返回通过/失败）
   - 提示：自动检查 `embargo_days >= max_lag`，若不满足显示警告，防止信息泄露

3. 特征工程（Feature Engineering）

   - 多选：滞后 `lags`（例如 1,2,5,10,21…）
   - 多选：滑动窗口/滚动统计（rolling windows，例如 5,10,21,63）
   - 数值输入：EWMA span
   - 目标特征数：滑块或数值输入（`TARGET_FEATURES`，建议范围 50–500，常用 250）
   - SHAP：是否使用 SHAP 做特征选择 + SHAP 采样大小
   - 按钮：生成特征预览（在中区展示前若干行样例）

4. 模型选择器

   - 选项：LightGBM / 神经网络（LSTM / MLP）/ PPO / 集成
   - 根据选择动态展开对应模型面板，隐藏与显示相关参数

5. LightGBM 面板（展开时）

   - 输入项：学习率、叶子数、最小子样本数、特征采样比例、bagging 比例与频率、L1/L2 正则、迭代轮数、早停轮数等
   - 开关：是否启用 Optuna（超参数搜索），以及试验次数
   - 按钮：运行 CV 训练（触发中区的训练流程）

6. 神经网络面板（LSTM / MLP）

   - 模型类型选择：LSTM / MLP
   - 输入：序列长度 `seq_len`、hidden_dim、层数、dropout
   - 训练相关：预训练/微调轮数、学习率、批大小、换手正则等
   - 开关：是否使用双精度用于几何均值计算
   - 按钮：运行 NN 训练（可拆分小任务并显示进度）

7. PPO 面板（展开时）

   - 输入：窗口长度、回合长度、换手成本
   - PPO 超参数：学习率、更新次数、每次更新步数、轮数、小批量大小、截断系数、熵系数
   - 高级：是否冻结基础网络若干更新
   - 按钮：开始 PPO 训练（会话式执行，输出日志）

8. 后处理 / 映射（Postprocessing / Mapping）

   - 开关：是否应用 Isotonic 校准及越界处理策略
   - 映射形式下拉：线性 / Sigmoid / Tanh / 自定义（若自定义则提供参数滑条或小型表达式编辑）
   - CMA-ES 优化器（折叠隐藏）：是否启用、迭代次数、sigma、初始候选、换手惩罚等
   - 波动率缩放：开关 + 窗口长度、目标波动率比率、裁剪范围

9. 预设与保存

   - 预设下拉：保守 / 平衡 / 激进（各预设应说明修改了哪些参数）
   - 按钮：保存配置 / 加载配置 / 恢复默认

行为提示：

   - 修改 CV 或特征参数应标记“模型产物已失效”，提示需重新训练
   - 当数据或特征设置变化时，禁用模型运行按钮直到重新生成特征

## 中间主区：运行控制与可视化（主交互区）

建议使用选项卡分区：概览（Overview）/ 训练（Training）/ 诊断（Diagnostics）/ 比较（Compare）。

### 概览（Overview）

- 数据预览表（前后 5 行）、特征数量、日期范围等摘要信息
- 小型 KPI 卡片：样本数、特征数、含缺失行数、最近一次 CV 的平均调整后夏普、最近运行时间
- 按钮：快速烟雾测试（Quick smoke test）——在小样本上快速跑完整 pipeline（通常 10s–2min）

### 训练区（Training）

- 展示当前选定模型与关键超参摘要（只读）
- 控件：主要操作按钮 `运行`（Run）、`停止`（Stop）、可选“后台运行”复选框（若不支持则灰化）
- 进度条与日志：显示不确定进度或折叠的折次进度，不建议显示 ETA（除非能可靠估算）
- 实时控制台输出：显示 stdout、每轮信息、每折结果
- 侧边摘要：最近指标快照（折次分数、均值±标准差、最佳验证分数）

提示：对于高耗操作（Optuna、CMA-ES、PPO）弹出确认对话，提示“此操作可能耗时，是否继续？”，并展示估算工作量指标（例如：#folds × 模型复杂度 × 迭代数），而非具体时间。

### 诊断区（Diagnostics）

交互图表（支持导出 / 缩放 / 悬浮提示）：

- 时序图：仓位随时间变化、策略与市场累计收益对比、滚动 Sharpe（带窗口滑块）
- 分布图：仓位直方图、换手直方图与序列图
- CV 结果表：每折的 adj_sharpe、训练迭代、最佳迭代、验证损失，并可查看该折的特征重要性 / SHAP 汇总 / 特征漂移
- LightGBM 的特征重要性面板：显示前 30 个特征并支持导出 CSV

额外工具：`显示评分调试（Show score debug）` 按钮，展示上一次运行中 score() 使用的中间统计量（例如 mu_strat、mu_market、sigma_strat、sigma_market、vol_penalty、return_penalty），便于排查 ddof 等差异。

---

## 右侧侧栏：运行历史、产物与比较

- 运行历史（Run History）：按时间列出历史运行，展示时间戳、模型类型、主配置摘要、调整后夏普、运行状态（成功/失败）、产物路径；可展开查看详细信息
- 比较（Compare）：选择最多 2–3 次运行并排比较仓位、累计收益与 CV 得分
- 产物（Artifacts）：快速访问已保存模型文件、配置、CMA-ES 参数、校准 pickle 等
- 快速操作：下载提交 CSV、重跑选中配置、将运行提升为 baseline
- 警告/告警：以红色显示验证警告（例如 embargo 不匹配、可能存在泄露）

---

## 底部：运行控制栏与控制台

- 在此处重复放置大按钮 `运行` / `停止` 以便快速操作
- 进度条：显示折次级别的进度（例如 “Fold 3/10 — training...”），不显示不可靠的 ETA
- 控制台：可滚动、支持复制 / 保存日志按钮
- 复选框：运行时自动保存配置
- 运行备注：允许为当前运行添加自由文本备注（metadata）

---

## 交互细节与 UX 约定（重要）

- 即时校验：当用户修改 CV、滞后或滚动窗口参数时，自动进行快速一致性检查（小样本）并在界面上以通过/失败方式展示结果，耗时很短
- 不可逆或高耗操作：Optuna、CMA-ES、PPO 等在点击后均弹出确认对话，并要求用户勾选“我已阅读并理解”才能继续
- 预设（Presets）：一键加载“保守 / 平衡 / 激进”预设，并在 tooltip 中列出该预设修改了哪些参数
- 配置导出/导入：支持 JSON 格式，便于复现与分享
- 安全校验：在生成最终提交文件前必须通过“验证分数一致性”步骤（脚本将官方分数与本地 score() 做对比），若不通过则阻止提交并展示差异报告
- 对比视图：右侧允许同时叠加最多 3 次运行的关键曲线（累计收益、调整后夏普等）
- 可访问性：数值控件同时支持键盘输入与滑块，控件拥有合理的最小/最大值与步长

---

## 示例工作流（典型操作）

1. 选择 LightGBM 模式 -> 左侧特征工程设置 `TARGET_FEATURES=375`，选择 Purged K-Fold 并设置 `n_splits=10`、`embargo_days=23` -> 点击“运行 CV 训练”。
2. CV 完成后在中区查看 CV 表与特征重要性 -> 选择 Isotonic 校准并打开 CMA-ES（迭代次数=150）-> 点击“运行 CMA-ES”。
3. 在右侧运行历史中选择最优运行，与基线比较；若满意则点击“下载提交 CSV”。

## 小结（优先级提示）

- 左侧：把 CV、TARGET_FEATURES、LightGBM 核心参数与映射 / CMA-ES 放在上层位置，便于频繁调整
- 中区：运行按钮、实时日志与关键图表（仓位/累计收益/滚动 Sharpe）是用户最常查看的内容
- 右侧：历史运行、对比与产物便于复现与回退
- 高耗操作（Optuna / CMA-ES / PPO）应放在高级折叠面板，并在启动前弹出确认对话
